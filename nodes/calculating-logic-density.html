How to Calculate Logic Density in Structured Datasets

As dataset pipelines transition from raw text dumps to deterministic training modules, logic density calculation has emerged as a key engineering metric [27]. Logic density measures the ratio of logical conditionals, syntax variables, and structural schemas to plain text volume [24]. For modern AI development teams, understanding logic density ensures that dataset tokens are rich in algorithmic reasoning structures and free of low-utility noise [24].

The Logic Density Formula

To calculate logic density programmatically, we map the ratio of logical decision nodes within an Abstract Syntax Tree (AST) to the total character or token volume:

Logic Density (LD) = (C_nodes + F_nodes + O_nodes) / Total_Tokens

Where:

  • C_nodes (Conditional Nodes): If/else pathways, ternary operators, case switch statements.
  • F_nodes (Functional Nodes): Function definitions, lambda expressions, map-reduce loops.
  • O_nodes (Operator Nodes): Logical AND/OR gates, arithmetic assertions, index constraints.

Code Evaluator AST Parser Blueprint

The following standard vanilla PostgreSQL function shows how to track and query logic density scores directly inside a Supabase backend database, allowing real-time monitoring of validator outputs: 


-- Supabase Row-Level Security Compliant Query
-- Calculates Average Logic Density by Tier Index
CREATE OR REPLACE FUNCTION get_logic_density_averages()
RETURNS TABLE (
  competency_tier TEXT,
  avg_density NUMERIC,
  active_nodes INT
) AS $$
BEGIN
  RETURN QUERY
  SELECT 
    role_tier AS competency_tier,
    ROUND(AVG(logic_density_score)::numeric, 4) AS avg_density,
    COUNT(id)::int AS active_nodes
  FROM system_nodes
  WHERE is_anonymized = true
  GROUP BY role_tier
  ORDER BY avg_density DESC;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
